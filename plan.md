## 2️⃣ 왜 이 주제를 하려고 하나? (문제 정의)

현재 아동·청소년의 온라인 위험 감지 연구에는 다음과 같은 문제가 있습니다.

1. **중앙 서버 기반 감시**
   - 메시지, 대화, 콘텐츠 원문을 서버로 수집
   - 프라이버시 침해 및 윤리적 문제

2. **절대적 위험 판정**
   - 특정 키워드(자살 등)를 사용하면 곧바로 위험으로 간주
   - 오탐, 낙인 문제 발생

3. **개인 변화와 집단 비교의 혼동**
   - 또래보다 다르다는 이유만으로 문제시하거나
   - 개인의 일시적 변화에 과도한 개입 발생

👉 **본 연구는 ‘위험 판정’이 아니라  
‘의미 있는 변화(change)’에 초점을 둡니다.**

---

## 3️⃣ 이 연구는 무엇을 보지 않는가? (윤리 포인트)

본 연구는 다음을 **의도적으로 배제**합니다.

- ❌ 카카오톡·메신저 대화
- ❌ 타인과의 상호작용 텍스트
- ❌ 원문 데이터의 서버 전송
- ❌ “이 아이는 위험하다”는 중앙 판정

👉 대신 **개인이 혼자 남기는 검색 쿼리**만을 사용합니다.  
검색은 타인 침해가 없고, 개인의 내적 상태 변화를 잘 반영하는 신호입니다.

# 4️⃣ 전체 시스템 개요 (구현 포함)

---

# 🔹 1. 구현 방식: 브라우저 확장 프로그램 기반

### ✔ 1단계: 크롬 확장 프로그램 개발

- 브라우저 주소창 검색 쿼리 감지
- 네이버/구글 등 검색 입력 이벤트 감지
- 검색어 문자열을 로컬 메모리에서만 처리

📌 이 단계에서:

- 검색어는 **절대 서버로 원문 전송하지 않음**

---

# 🔹 2. 로컬(개인 기기)에서 하는 일

### ① 검색 쿼리 수집 (로컬)

예:

"공부 너무 힘들다"  
"잠이 안 온다"  
"죽고 싶을 때 도움"

---

### ② BGE-M3-ko 기반 임베딩

- 쿼리를 BGE-M3-ko 모델로 임베딩
- 벡터 공간에서 의미 표현 획득

---

### ③ 카테고리 점수 계산 (B안 방식)

카테고리 예:

- Anxiety
- Depression
- Suicidal
- Normal

각 카테고리별 프로토타입 문장 세트와  
코사인 유사도 계산:

score_distress = max cosine(query, proto_distress_j)

→ 문장 단위 카테고리 점수 생성

---

### ④ 시간 윈도우 집계 (예: 7일)

윈도우 요약 벡터 xₜ 생성:

x_t = {  
}

---

### ⑤ 개인 기준선(Self-baseline) 계산

- 과거 4~6주 평균과 비교
- 추세(slope) + 지속성 판단
- 단발 이벤트 필터링
- 워밍업 기간 설정

---

### ⑥ 로컬 판단

조건:

- 개인 변화 발생
- 또래 기준 대비 드묾
- 일정 기간 지속

→ 아이에게만 지원 리소스 제안

📌 부모 통보, 서버 통보 없음

---

# 🔹 3. 서버에서 하는 일

### 서버는 절대 하지 않는 것

- 개인 원문 분석 ❌
- 개인 위험 판정 ❌
- 개별 사용자 추적 ❌

---

### 서버가 받는 데이터

- 원문 ❌
- 개별 점수 ❌
- 판단 결과 ❌

✔ 오직:

윈도우 요약 통계 (x_t)  
또는 sum / sumsq / count

---

### 서버에서 수행하는 작업

1. 같은 연령 코호트 집계
2. 평균(μ), 분산(σ) 계산
3. robust aggregation 적용
4. 신뢰도 기반 가중치 업데이트

---

### 서버가 내려주는 것

NormPack {  
 Anxiety
 Depression
 Suicidal
 Normal
}

→ 개인 기준이 아니라 **집단 기준**

# 5️⃣ 핵심 아이디어: 두 가지 기준의 분리

## ① 개인 기준선 (Self-baseline)

- 이 아이의 과거 대비 변화 감지
- 상승 추세 + 지속성

## ② 또래 기준 (Peer norm)

- 같은 연령 집단의 정상 분포
- z-score 기반 드묾 판단

📌 두 조건이 동시에 만족될 때만 개입
